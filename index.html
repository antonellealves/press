01 — O que mudou

De requisições fixas
para consumo por tokens

O modelo antigo abstraia o custo real: toda interação valia "uma requisição", independente de complexidade. O novo expõe a economia real do processamento de IA.

até maio 2026

Premium Requests

Uma pergunta simples e uma sessão de agente de horas consumiam exatamente a mesma coisa. Fatura fixa todo mês. Fácil de orçar, mas economicamente insustentável para a Microsoft.

a partir de jun 2026

GitHub AI Credits

Cada token processado — tanto no envio quanto na geração de resposta — tem um custo baseado no modelo utilizado. A fatura escala com o que você realmente usa.

Contexto automático: o Copilot inclui automaticamente em cada requisição o arquivo ativo, código selecionado, arquivos próximos e conteúdo relevante do repositório — além de arquivos como .github/copilot-instructions.md e o histórico de conversas. 05 — Como economizar

Estratégias para
controlar o consumo

Ajustes de hábito e configuração que podem reduzir significativamente o consumo de créditos sem abrir mão da produtividade.

01

Referencie arquivos específicos, não o repositório inteiro

Configure o Copilot para apontar só os módulos relevantes para cada tarefa. Reduzir o contexto de 100.000 para 20.000 tokens corta os custos de input em até 80%.

02

Agrupe perguntas em uma única sessão de chat

Cada sessão nova carrega overhead de system prompt e inicialização de contexto. Cinco perguntas numa sessão custam menos que cinco sessões individuais com uma pergunta cada.

03

Escolha o modelo certo para cada tarefa

Modelos leves (como GPT-4o-mini ou Claude Haiku) custam tokens muito mais baratos. Reserve os modelos frontier (Opus, GPT-4.1) para tarefas que realmente demandam raciocínio complexo.

04

Revise e enxugue o .github/copilot-instructions.md

Esse arquivo é enviado como input em toda interação. Mantenha-o conciso. Cada linha desnecessária vira custo em todas as requisições do time.

05

Monitore antes de a fatura fechar

O dashboard de billing do GitHub mostra consumo por modelo em tempo real. Acesse em github.com → Settings → Billing & plans e configure alertas antes do fim do ciclo.
